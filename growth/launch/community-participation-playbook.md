## 0. The governing rule (same as community-plan, restated because it governs everything)

Communities define spam by **behavior, not intent**. An account whose history is mostly links to its own thing is a spammer, period. So this is a ratio, not a campaign.

**Contribute ≫ promote — ~9:1.** Nine genuinely helpful actions (that never mention trovex) for every one that does. Be a person who knows the problem, not an account that drops a link and leaves.

Everything below is downstream of that ratio.

---

## 1. Why participation earns *citation*, not just upvotes (the GEO link)

A vanity-free reason to do this well: **AI engines cite what humans cite.** When you answer the canonical "how do I stop my agent rereading docs / cut agent token cost" question thoroughly in a place that gets indexed (a high-rep HN comment, a GitHub discussion, a Stack Overflow answer, a well-received Reddit post), that content becomes a source ChatGPT / Perplexity / Google AI Overviews can quote when someone asks the same thing later. (geo-lead owns the on-site GEO; this is the off-site half.)

So the unit of value is **a durable, genuinely-useful answer to a real recurring question** — not a link. Links decay; a great answer keeps getting found and cited.

Corollary: write contributions to be *self-contained and quotable*. A complete answer that helps even if trovex didn't exist is exactly what gets cited — and trovex is the credible footnote, not the point.

## 3. The contribution ladder (kinds of value, low → high effort)

Climb it over time; the higher rungs earn the most durable awareness + citation.

1. **Answer a question well.** The base unit. Find a real thread (token cost, agents rereading docs, CLAUDE.md going stale, RAG-for-code) and give a complete, honest answer. Mention trovex only on direct fit, plainly.
2. **File a good bug / thoughtful issue** on a client or framework you actually use. Pure goodwill; makes you a known name.
3. **Ship a real PR / integration gift.** A working "trovex + <client>" config, a Goose extension, a Mastra example (see partnerships.md). Code is the highest-trust contribution in dev communities.
4. **Write a self-contained, quotable explainer.** "How I measured what my coding agent spends rereading .md files (and cut it ~60%)" — method + real numbers, trovex as the tool you used, not the headline. This is the citation engine (§1).
5. **Give a talk / go on a podcast / guest post.** Earned media — handed off to the earned-media targets doc + outreach.md. Highest reach, requires standing first.

Rule: never start at rung 4–5 in a venue where you have no standing. Earn it on rungs 1–3 first.

## 4. Cadence (ongoing habit, not a sprint)

Not a 30-day push — a sustainable rhythm a human can keep.

- **Weekly:** answer 3–5 real questions across venues (rung 1). Zero promo most weeks.
- **Monthly:** one rung-3 gift (a PR/integration/example) where it genuinely fits.
- **Quarterly:** one rung-4 explainer in the single best-fit venue, written to be quotable; plus consider one rung-5 earned-media move once standing is real.
- **Always:** reply to everyone who engages with anything you posted; concede real limits (small doc sets, repo-dependent savings). Turn bugs/feedback into GitHub issues — that's real proof.

> The launch window (community-plan.md) is a temporary intensification of this rhythm, not a different activity.

---

## 5. Guardrails (hard stops — same family as community-plan + outreach)

- No alt accounts, no vote-begging, no planted "has anyone tried trovex?" sockpuppets.
- No copy-paste across venues; no same link in five places same day.
- No DMing strangers cold with the link.
- No arguing with skeptics or getting defensive — concede limits, move on.
- No consulting pitch in any community surface. If asked about team help: a plain "I do that kind of consulting, happy to talk — DM me," only when asked. (north-star door stays low-key.)
- No "Synergix" in any post or profile (brand rule); the repo URL identifier is the only place that string appears.
- No fabricated experience or numbers. Didn't measure it → don't claim it. Real ~60%, repo-dependent, always.
- lowercase `trovex`; no banned words (revolutionary/seamless/supercharge/unlock/AI-powered).

---

## 6. How we know it worked (north-star aligned, not vanity)

- **Real installs/indexes** traceable to a thread (activation) + bugs/feature requests filed as GitHub issues — beats upvotes.
- **Standing**: a recognized helpful name in r/mcp + the MCP Discords, so future contributions land warmer.
- **Citation surface**: our answers start showing up when AI engines / search answer the recurring questions (the GEO payoff; coordinate with geo-lead).
- **The consulting door**: the occasional "we run a team, can you help" DM. One of those beats a hundred non-installing upvotes.
