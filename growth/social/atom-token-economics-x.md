## Thread

**1/ hook**
> one coding agent rereading your docs is a rounding error. ~700 tokens to find the current
> file. who cares.
>
> now multiply: every lookup × every session × every agent × every teammate. that rounding
> error is a monthly line item. the math 👇

**2/**
> the cost is invisible because you meet it one lookup at a time, and one lookup is cheap. it
> never arrives as a bill. it arrives as a small tax on every interaction — and you notice it
> at month end as "why is our spend creeping up."

**3/ the multiplication**
> 5 agents × 30 doc lookups/day × ~440 wasted tokens (reading files they discard) × 20 days.
> that's the slice you're paying to re-derive answers you already paid for last week.

**4/**
> money isn't the whole cost. every stale file the agent reads to "check" pollutes the context
> window and nudges it toward the older, wrong answer. you pay more to get worse.

**5/ the fix compounds the other way**
> serve one canonical doc per query (path:line + freshness, section-level) instead of letting
> each agent reread to guess. ~60% fewer tokens per doc lookup — and the saving multiplies by
> the same fleet the waste did.
