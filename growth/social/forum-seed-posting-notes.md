# Forum seeds — posting notes & rules (READ BEFORE POSTING)

> These are drafts. A **human** posts them, from a **real account with genuine history**.
> Never astroturf ("I'm just a happy user"), never post the same text to multiple
> communities, never post the product as the headline. If you're involved with trovex,
> disclose it when it's relevant or asked. The point is qualified reach, not a spike.

## The non-negotiables (all communities)

1. **Value-first.** Each post must be useful even if the reader ignores trovex. The tool is
   ONE option mentioned mid-body, never the title, never the lede.
2. **One real number only.** ~60% fewer tokens on doc lookups (or your own measured receipt).
   No fabricated stars, users, testimonials, or logos — pre-launch, zero customers.
3. **No banned words.** revolutionary / seamless / supercharge / unlock / "AI-powered".
   Lowercase `trovex`. Write from the user's side.
4. **Disclose involvement** when relevant or asked. Honesty is the whole strategy here.

## Per-community rules

### r/LocalLLaMA
- Angle: local-first (SQLite + ONNX, no cloud/keys), token frugality, hybrid retrieval.
- Read the sub's self-promo rule before posting; some weeks have a dedicated thread.
- Lead with the problem/discussion; end with real questions. Be present to reply.

### r/ChatGPTCoding
- Angle: agent (Claude Code / Cursor) rereading the repo, the token bill.
- **~9:1 rule** — for every promotional mention, have ~9 genuine non-promo contributions in
  the sub first. Don't let this be your first-ever post.
- Flair appropriately if the sub requires it.

### Lobsters
- **Submit a technical write-up, NOT the product.** A product announcement / landing page
  reads as a write-only promo channel and violates norms.
- **Check "authored by submitter"** — mandatory if you're involved.
- **Self-promo must stay a small fraction (<~25%)** of your total stories+comments. If your
  history is thin, build it first or don't submit.
- Tag accurately from the live tag list. The comment section is the value — answer honestly.

## Cadence (don't fire all at once)

- Stagger across days/weeks, not the same day. Identical-timing cross-posts look coordinated.
- Tailor each post to the community (these drafts already are) — never copy-paste one body
  into three subs.
- Sequence suggestion: write the technical blog post first (content-lead) → Lobsters submits
  it → Reddit posts reference the lived problem, link the repo mid-body → reply to comments
  for the first ~hour (that's where the reach compounds).

## What success looks like
Qualified devs trying `uv run trovex index <repo>` and seeing their own savings number — and
the occasional team lead reaching out about running agents at scale. Not upvote count.
